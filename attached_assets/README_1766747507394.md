## рџљЂ Features

### Core Capabilities
- **Real-time Device Management**: Monitor and manage ESP32 devices with live status updates
- **Firmware Version Control**: Upload, manage, and deploy multiple firmware versions
- **Batch OTA Deployments**: Deploy updates to multiple devices simultaneously with queue management
- **Transaction Rollback**: Automatic rollback on failed updates with state preservation
- **Comprehensive Logging**: Winston-based logging with file rotation and viewing
- **Input Validation**: Zod schema validation on all API endpoints
- **Update Queue System**: Prevents concurrent updates with p-queue (max 5 simultaneous)
- **Security Hardening**: Rate limiting, directory traversal protection, MAC validation

### Device Management
- Real-time device status tracking (online, offline, pending, updated, failed)
- MAC-based device identification for ESP32 compatibility
- Device naming and organization
- Last seen timestamps and OTA check tracking
- Device creation, update, and deletion with validation

### Firmware Management
- Multi-version firmware support with SHA256 checksums
- File size and type validation (.bin and .hex files only, max 16MB)
- Automatic checksum calculation for integrity verification
- Download count tracking for analytics
- Firmware deletion with file cleanup

### OTA Deployments
- Queue-based deployment system preventing race conditions
- Per-device deployment tracking and status
- Automatic version history maintenance
- One-click rollback to previous firmware version
- Real-time deployment activity monitoring
- Duplicate update prevention (same version within 5 minutes)

### Production-Ready Features
- **Atomic Operations**: All database updates are atomic
- **Error Boundaries**: Try-catch blocks on all async handlers
- **Comprehensive Logging**: Every operation logged with Winston
- **Health Monitoring**: Health check and status endpoints
- **Rate Limiting**: Protection against abuse on critical endpoints
- **Transaction Support**: State capture and rollback on failures

## рџ› пёЏ Installation

### 1. Clone Repository

```bash
git clone https://github.com/satya1541/ota.git
cd ota
npm install
```

### 2. Database Configuration

Create a `.env` file in the root directory:

```env
# Database Configuration
DATABASE_URL=mysql://username:password@host:port/database_name

# Example:
DATABASE_URL=mysql://satya:satya123@15.206.156.197:3306/ota_db

# Server Configuration
PORT=5000
NODE_ENV=development
```

### 3. Database Setup

Push the Drizzle schema to your database:

```bash
npm run db:push
```

This creates the following tables:
- `devices` - Device registration and status
- `firmware` - Firmware versions and metadata
- `device_logs` - OTA event logging
- `users` - Authentication (if enabled)

### 4. Create Firmware Directory

The firmware upload directory will be created automatically on first run at:
```
server/firmware/
```

### ESP32 Device Routes

#### `GET /ota/check`
Check if firmware update is available for a device.

**Query Parameters:**
- `mac`: Device MAC address (required)

**Validation:**
- MAC address format: XX:XX:XX:XX:XX:XX

**Rate Limit:** 10 requests per minute per device

**Response (Update Available):**
```json
{
  "updateAvailable": true,
  "version": "v1.2.0",
  "url": "http://yourserver:5000/firmware/firmware_v1.2.0_1735201234567.bin",
  "checksum": "abc123def456789...",
  "size": 1048576,
  "description": "WiFi stability improvements"
}
```

**Response (No Update):**
```json
{
  "updateAvailable": false,
  "currentVersion": "v1.2.0"
}
```

**Features:**
- Automatic device status update to "online"
- Sets `otaStatus` to "updating" when update available
- Updates `lastOtaCheck` timestamp
- Logs unregistered device attempts

#### `POST /ota/report`
Report OTA update result from device.

**Request Body:**
```json
{
  "mac": "AA:BB:CC:DD:EE:FF",
  "success": true,
  "version": "v1.2.0",
  "message": "Update successful"
}
```

**Validation:**
- `mac`: Required, valid MAC format
- `success`: Required, boolean
- `version`: Required, semantic version
- `message`: Optional, max 500 characters

**Response:**
```json
{
  "success": true,
  "message": "Status updated successfully"
}
```

**Features:**
- Atomic status updates
- On success: updates `currentVersion`, sets `otaStatus` to "updated"
- On failure: sets `otaStatus` to "failed"
- Updates device status to "online"
- Comprehensive logging with timestamp

## рџ“± ESP32 Integration

### ESP32 Client Code Example

Complete Arduino implementation for ESP32 devices:

```cpp
#include <WiFi.h>
#include <HTTPClient.h>
#include <Update.h>
#include <ArduinoJson.h>

// Configuration
const char* WIFI_SSID = "YOUR_WIFI_SSID";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
const char* OTA_SERVER = "http://192.168.1.100:5000";
const char* CURRENT_VERSION = "v1.0.0";
const unsigned long CHECK_INTERVAL = 300000; // 5 minutes

String macAddress;
unsigned long lastCheck = 0;

void setup() {
  Serial.begin(115200);
  
  // Get MAC address
  macAddress = WiFi.macAddress();
  macAddress.replace(":", "");
  Serial.println("MAC: " + macAddress);
  
  // Connect to WiFi
  connectWiFi();
}

void loop() {
  if (WiFi.status() != WL_CONNECTED) {
    connectWiFi();
  }
  
  if (millis() - lastCheck > CHECK_INTERVAL) {
    checkForUpdate();
    lastCheck = millis();
  }
  
  // Your main application code here
  delay(1000);
}

void connectWiFi() {
  Serial.println("Connecting to WiFi...");
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  
  int attempts = 0;
  while (WiFi.status() != WL_CONNECTED && attempts < 20) {
    delay(500);
    Serial.print(".");
    attempts++;
  }
  
  if (WiFi.status() == WL_CONNECTED) {
    Serial.println("\nWiFi Connected!");
    Serial.println("IP: " + WiFi.localIP().toString());
  }
}

void checkForUpdate() {
  if (WiFi.status() != WL_CONNECTED) return;
  
  HTTPClient http;
  String url = String(OTA_SERVER) + "/ota/check?mac=" + macAddress;
  
  Serial.println("Checking for updates...");
  http.begin(url);
  
  int httpCode = http.GET();
  
  if (httpCode == 200) {
    String payload = http.getString();
    DynamicJsonDocument doc(1024);
    deserializeJson(doc, payload);
    
    bool updateAvailable = doc["updateAvailable"];
    
    if (updateAvailable) {
      String version = doc["version"];
      String downloadUrl = doc["url"];
      String checksum = doc["checksum"];
      
      Serial.println("Update available: " + version);
      performOTA(downloadUrl, checksum, version);
    } else {
      Serial.println("No update available");
    }
  }
  
  http.end();
}

void performOTA(String url, String expectedChecksum, String version) {
  HTTPClient http;
  http.begin(url);
  
  int httpCode = http.GET();
  
  if (httpCode == 200) {
    int contentLength = http.getSize();
    
    if (contentLength > 0) {
      bool canBegin = Update.begin(contentLength);
      
      if (canBegin) {
        Serial.println("Starting OTA update...");
        WiFiClient * stream = http.getStreamPtr();
        size_t written = Update.writeStream(*stream);
        
        if (written == contentLength) {
          Serial.println("Written : " + String(written) + " successfully");
        }
        
        if (Update.end()) {
          if (Update.isFinished()) {
            Serial.println("Update successfully completed!");
            reportStatus(true, version, "Update successful");
            delay(1000);
            ESP.restart();
          }
        } else {
          Serial.println("Update failed: " + String(Update.getError()));
          reportStatus(false, version, "Update failed: " + String(Update.getError()));
        }
      } else {
        reportStatus(false, version, "Not enough space for update");
      }
    }
  } else {
    reportStatus(false, version, "Download failed: " + String(httpCode));
  }
  
  http.end();
}

void reportStatus(bool success, String version, String message) {
  if (WiFi.status() != WL_CONNECTED) return;
  
  HTTPClient http;
  String url = String(OTA_SERVER) + "/ota/report";
  
  http.begin(url);
  http.addHeader("Content-Type", "application/json");
  
  DynamicJsonDocument doc(512);
  doc["mac"] = macAddress;
  doc["success"] = success;
  doc["version"] = version;
  doc["message"] = message;
  
  String payload;
  serializeJson(doc, payload);
  
  int httpCode = http.POST(payload);
  Serial.println("Report sent: " + String(httpCode));
  
  http.end();
}
```

**Required Libraries:**
- ArduinoJson (by Benoit Blanchon)
- Built-in: WiFi, HTTPClient, Update

**Installation:**
1. Install ArduinoJson library from Library Manager
2. Update WiFi credentials and server URL
3. Set current firmware version
4. Upload to ESP32

## рџ’ѕ Database Schema

### Table: `devices`
Stores device information and status.

| Column | Type | Description |
|--------|------|-------------|
| id | INT (PK) | Auto-increment ID |
| name | VARCHAR(100) | Device friendly name |
| macAddress | VARCHAR(17) UNIQUE | MAC address (XX:XX:XX:XX:XX:XX) |
| currentVersion | VARCHAR(20) | Installed firmware version |
| targetVersion | VARCHAR(20) | Version to update to |
| previousVersion | VARCHAR(20) | Previous version for rollback |
| status | ENUM | online, offline, pending, failed |
| otaStatus | ENUM | idle, pending, updating, updated, failed |
| lastSeen | TIMESTAMP | Last communication timestamp |
| lastOtaCheck | TIMESTAMP | Last OTA check timestamp |
| createdAt | TIMESTAMP | Device registration time |

**Indexes:**
- PRIMARY KEY (id)
- UNIQUE KEY (macAddress)
- INDEX (status)
- INDEX (otaStatus)

### Table: `firmware`
Stores firmware version metadata.

| Column | Type | Description |
|--------|------|-------------|
| id | INT (PK) | Auto-increment ID |
| version | VARCHAR(20) UNIQUE | Semantic version string |
| filename | VARCHAR(255) | Physical filename on disk |
| description | TEXT | Release notes |
| size | BIGINT | File size in bytes |
| checksum | VARCHAR(64) | SHA256 hash |
| downloadCount | INT | Download counter |
| uploadedAt | TIMESTAMP | Upload timestamp |

**Indexes:**
- PRIMARY KEY (id)
- UNIQUE KEY (version)
- INDEX (uploadedAt)

### Table: `device_logs`
Stores OTA activity logs.

| Column | Type | Description |
|--------|------|-------------|
| id | INT (PK) | Auto-increment ID |
| deviceId | INT | Foreign key to devices.id |
| macAddress | VARCHAR(17) | Device MAC address |
| action | VARCHAR(50) | check, deploy, report, rollback |
| status | VARCHAR(50) | success, failed, pending |
| message | TEXT | Log message |
| fromVersion | VARCHAR(20) | Source version |
| toVersion | VARCHAR(20) | Target version |
| createdAt | TIMESTAMP | Log entry timestamp |

**Indexes:**
- PRIMARY KEY (id)
- INDEX (deviceId)
- INDEX (macAddress)
- INDEX (action)
- INDEX (status)
- INDEX (createdAt)

### Table: `users` (Optional)
Stores user authentication data.

| Column | Type | Description |
|--------|------|-------------|
| id | INT (PK) | Auto-increment ID |
| username | VARCHAR(100) UNIQUE | Username |
| password | VARCHAR(255) | Hashed password |
| createdAt | TIMESTAMP | Account creation time |

## рџ”’ Security Features

### Input Validation
- **Zod Schemas**: All API inputs validated with Zod
- **MAC Address Validation**: Regex pattern matching (XX:XX:XX:XX:XX:XX)
- **Version Validation**: Semantic version format enforcement
- **File Type Validation**: Only .bin and .hex files accepted
- **File Size Limits**: Maximum 16MB per firmware file

### Rate Limiting
```typescript
// OTA check endpoint: 10 requests per minute per device
// Download endpoint: 30 requests per minute
```

### Security Measures
- **Directory Traversal Protection**: Path validation on file downloads
- **Atomic Database Operations**: All updates are atomic
- **Transaction Rollback**: Failed updates rollback to previous state
- **Error Boundaries**: Try-catch on all async handlers
- **IP Logging**: Security events logged with IP addresses
- **Session Management**: Express-session with secure settings

### Logging
- **Winston Logger**: File-based logging with rotation
- **Log Levels**: info, warn, error
- **Log Files**:
  - `server/logs/ota.log` - OTA-specific events
  - `server/logs/error.log` - Error events only
  - `server/logs/combined.log` - All events
- **Log Rotation**: Automatic rotation and compression

### Best Practices
1. Use HTTPS in production
2. Implement authentication middleware
3. Set strong database passwords
4. Use environment variables for secrets
5. Regular security audits
6. Keep dependencies updated

## рџђ› Troubleshooting

### Common Issues

#### Server Won't Start

**Problem:** Server fails to start or crashes immediately.

**Solutions:**
```bash
# Check database connection
npm run db:push

# Verify environment variables
cat .env  # Linux/macOS
type .env # Windows

# Check if port is already in use
# Windows
netstat -ano | findstr :5000
# Linux/macOS
lsof -i :5000

# Kill process using port
# Windows
taskkill /PID <PID> /F
# Linux/macOS
kill -9 <PID>

# Check Node.js version
node --version  # Should be 18+
```

#### Database Connection Errors

**Problem:** "Can't connect to MySQL server" or "Access denied"

**Solutions:**
1. Verify MySQL is running:
   ```bash
   # Linux
   sudo systemctl status mysql
   
   # Windows
   services.msc  # Check MySQL service
   ```

2. Test database connection:
   ```bash
   mysql -h 15.206.156.197 -u satya -p
   ```

3. Check DATABASE_URL format:
   ```env
   DATABASE_URL=mysql://username:password@host:port/database
   ```

4. Verify firewall rules allow connection to MySQL port (3306)

#### Firmware Upload Fails

**Problem:** "Failed to upload firmware" or "File not found"

**Solutions:**
1. Check firmware directory exists:
   ```bash
   ls -la server/firmware/  # Linux/macOS
   dir server\firmware\     # Windows
   ```

2. Verify file permissions:
   ```bash
   chmod 755 server/firmware/  # Linux/macOS
   ```

3. Check file size (max 16MB):
   ```bash
   ls -lh your-firmware.bin
   ```

4. Ensure file extension is .bin or .hex

#### ESP32 Can't Check for Updates

**Problem:** Device can't reach OTA server

**Solutions:**
1. Verify server is accessible:
   ```bash
   curl http://your-server:5000/health
   ```

2. Check ESP32 serial output for errors

3. Verify WiFi connection on ESP32

4. Check server firewall allows incoming connections:
   ```bash
   # Linux
   sudo ufw allow 5000/tcp
   
   # Windows Firewall
   netsh advfirewall firewall add rule name="OTA Server" dir=in action=allow protocol=TCP localport=5000
   ```

5. Test from ESP32's network:
   ```bash
   # From same network as ESP32
   curl "http://your-server:5000/ota/check?mac=AA:BB:CC:DD:EE:FF"
   ```

#### Update Queue Issues

**Problem:** Devices stuck in "updating" or queue not progressing

**Solutions:**
1. Check queue status:
   ```bash
   curl http://localhost:5000/api/queue/status
   ```

2. Check logs:
   ```bash
   # View OTA logs
   curl http://localhost:5000/api/logs/ota/view
   
   # Or check file directly
   cat server/logs/ota.log
   ```

3. Reset stuck devices:
   ```bash
   # Using API
   curl -X POST http://localhost:5000/api/reset \
     -H "Content-Type: application/json" \
     -d '{"deviceIds": [1, 2, 3]}'
   ```

4. Clear update history and restart:
   ```bash
   npm run dev
   ```

#### TypeScript Compilation Errors

**Problem:** Type errors during development

**Solutions:**
```bash
# Check for errors
npm run check

# Clean node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Update dependencies
npm update
```

#### Frontend Not Loading

**Problem:** Blank page or "Cannot GET /" error

**Solutions:**
1. Check if server is running:
   ```bash
   curl http://localhost:5000/health
   ```

2. Clear browser cache and hard reload (Ctrl+Shift+R)

3. Check browser console for errors (F12)

4. Verify build completed successfully:
   ```bash
   npm run build
   ```

5. Check server logs for errors

### Docker Deployment (Optional)

Create `Dockerfile`:
```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 5000

CMD ["npm", "start"]
```

Create `docker-compose.yml`:
```yaml
version: '3.8'

services:
  ota-server:
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=mysql://user:pass@mysql:3306/ota_db
      - NODE_ENV=production
    volumes:
      - ./server/firmware:/app/server/firmware
      - ./server/logs:/app/server/logs
    depends_on:
      - mysql
      
  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=rootpass
      - MYSQL_DATABASE=ota_db
      - MYSQL_USER=otauser
      - MYSQL_PASSWORD=otapass
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

volumes:
  mysql_data:
```

Deploy:
```bash
docker-compose up -d
```

## рџ“€ Monitoring & Maintenance

### Health Monitoring

Set up a cron job or monitoring service to check:
```bash
# Health check
curl http://your-server:5000/health

# System status
curl http://your-server:5000/api/status

# Queue status
curl http://your-server:5000/api/queue/status
```

### Log Management

Monitor log files:
```bash
# View recent OTA events
tail -f server/logs/ota.log

# View errors
tail -f server/logs/error.log

# Check log sizes
du -sh server/logs/*
```

Clean old logs periodically:
```bash
# Clear logs via API
curl -X DELETE http://localhost:5000/api/logs/ota
curl -X DELETE http://localhost:5000/api/logs/error
```

### Database Maintenance

Regular tasks:
```bash
# Backup database
mysqldump -u user -p ota_db > backup_$(date +%Y%m%d).sql

# Optimize tables
mysql -u user -p ota_db -e "OPTIMIZE TABLE devices, firmware, device_logs;"

# Check database size
mysql -u user -p ota_db -e "SELECT table_name, 
  ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)' 
  FROM information_schema.TABLES WHERE table_schema = 'ota_db';"
```

### Firmware Storage Cleanup

Monitor and clean unused firmware:
```bash
# Check firmware directory size
du -sh server/firmware/

# List old firmware files
find server/firmware -type f -mtime +30

# Delete firmware via API to clean both DB and file
curl -X DELETE http://localhost:5000/api/firmware/v1.0.0
```
